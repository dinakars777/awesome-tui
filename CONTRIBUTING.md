# Contributing to awesome-tui

Thank you for your interest in contributing! This guide will help you add your favorite TUI apps and frameworks.

## How to Contribute

### 1. Adding a TUI Application

**Requirements:**
- Must be a TUI (Text User Interface) application
- Should be actively maintained (updated in last 2 years)
- Must be open source or have a free version
- Should be useful and well-designed

**Steps:**
1. Fork this repository
2. Add your app to the appropriate category in README.md
3. Follow the format: `- [App Name](link) - Brief description`
4. Submit a pull request

**Example:**
```markdown
- [git-time-machine](https://github.com/dinakars777/git-time-machine) - Visual git reflog browser to undo mistakes
```

### 2. Adding a Framework

**Requirements:**
- Must be a library/framework for building TUIs
- Should have documentation
- Should have at least one example app
- Must be actively maintained

**Format:**
```markdown
- [Framework Name](link) - Brief description of what it does and key features
```

### 3. Adding a New Category

If you want to add a new category:
1. Make sure it doesn't overlap with existing categories
2. Should have at least 3-5 apps to justify a category
3. Add it to the table of contents
4. Place it in logical order in the README

## Quality Standards

### Good Descriptions

**Good:**
- "Visual git reflog browser to undo mistakes"
- "Fast terminal UI for git written in Rust"
- "Cross-platform system monitoring tool"

**Bad:**
- "The best git tool ever made"
- "A tool for git"
- "You should definitely use this"

### What to Include

- What the app does
- Key differentiator (if relevant)
- Language/framework (if notable)

### What NOT to Include

- Marketing language ("amazing", "revolutionary")
- Personal opinions ("my favorite")
- Version numbers (they change)
- Installation instructions (link to repo)

## Pull Request Process

1. **Check for duplicates** - Search the README first
2. **One app per PR** - Makes review easier
3. **Test the link** - Make sure it works
4. **Follow the format** - Match existing entries
5. **Write a clear PR description**:
   ```markdown
   ## What
   Adding [App Name] to [Category]

   ## Why
   [Brief explanation of why this app is awesome]

   ## Checklist
   - [ ] App is actively maintained
   - [ ] Link works
   - [ ] Description is concise
   - [ ] Added to correct category
   ```

## Categories Explained

### Development Tools
Tools for software development (IDEs, debuggers, build tools)

### Git & Version Control
Git clients, diff viewers, merge tools

### System Monitoring
Resource monitors, process viewers, system stats

### Productivity
Task managers, calendars, note-taking apps

### File Management
File browsers, file managers, directory navigation

### Databases
Database clients, query tools, admin interfaces

### Networking
Network tools, ping, traceroute, packet analyzers

### Media
Music players, video players, media libraries

### Games
Games that run in the terminal

### Fun
Entertainment, screensavers, ASCII art

## Review Process

1. Maintainers will review your PR within 48 hours
2. May ask for changes or clarifications
3. Once approved, your contribution will be merged
4. You'll be added to the contributors list!

## Code of Conduct

### Be Respectful
- Treat all contributors with respect
- Be constructive in feedback
- Help others improve their contributions

### Be Collaborative
- Share knowledge
- Help review other PRs
- Suggest improvements

### Be Inclusive
- Welcome newcomers
- Explain technical terms
- Be patient with questions

## Questions?

- Open an issue with the `question` label
- Check existing issues first
- Be specific about what you need help with

## Recognition

All contributors will be:
- Listed in the README (if they want)
- Credited in release notes
- Part of the awesome-tui community!
